1- INTERROGER LA BASE

L'interrogation de la base se fait avec le mot clef SELECT qui permet de sélectionner les données que l'on veut consulter:

A- Sélectionner les données:

1- La clause SELECT (permet de récuperer des données contenues dans une ou plusieurs tables)

selectionner toutes les colonnes de la table:

SELECT *
FROM nom_table;


sélectionner seulement certaines colonnes:

SELECT nom_colonne1, nom_colonne2, ...
FROM nom_table;


2- La  clause DISTINCT (permet d'éliminer les doublons)

SELECT DISTINCT nom_colonne1
FROM nom_table;


3-  La clause WHERE ( restreint les lignes sélectionnées apr une condition)

SELECT *
FROM nom_table
WHERE condition1 AND condition2 ...;


4- Les opérateurs de comparaison

Les conditions sont posées grâce aux opérateurs suivants:
                                =      égal
                                !=    différent
                                <      inférieur
                                >      supérieur
                                <=   inférieur ou égal
                                >=   supérieur ou égal

SELECT *
FROM nom_table
WHERE nom_champ1 > nom_champ2;


5- BETWEEN ( compris entre )

SELECT * FROM nom_table
WHERE nom_champ1
BETWEEN expression1 and expression2;


6- IN compris dans la liste de choix

SELECT * FROM  nom_table
WHERE nom_champ1 IN ( expression1, expression2, ...);


7- LIKE permet de comparer les chaines de caractères

SELECT * from nom_table
WHERE nom_champ1 LIKE 'chaine' ;
                                où chaine peut contenir des caractères jokers:
                                      _ remplace 1 caractère
SELECT * FROM nom_table
WHERE nom_champ LIKE 'ch_ine' ;


                                      % remplace une chaine de caractère de longueur quelconque.

SELECT * FROM nom_table
WHERE nom_champ1 LIKE  'c%' ;


8- NULL permet de savoir si le champ a été saisi ou nom

SELECT * FROM nom_table
WHERE nom_champ1 IS NULL AND champ2 IS NOT NULL ;


9- Les opérateurs logiques
 

                         AND:  ET logique où la condition1 et la condition2 doivent être vérifiées

SELECT * FROM table
WHERE condition1 AND condition2;


                         OR: OU logique où il suffit que l'une des conditions au moins soit vérifiée

SELECT * FROM table
WHERE  condition1 OR condition2;


                              NOT: NON logique qui permet de tester le contraire d'une condition

SELECT * FROM nom_table
WHERE nom_champ1 NOT IN (expression1, expression2) ;


10- Les jointures permettent de faire une requête sur plusieurs tables

SELECT nom_champ1, nom_champ2
FROM nom_table1, nom_table2
WHERE  nom_table1.nom_champ3 = nom_table2.nom_champ3;


11- Les requêtes imbriquées: elles peuvent se faire sur une ou plusieurs tables

SELECT nom_champ1, nom_champ2
FROM nom_table1
WHERE  nom_champ3 IN (SELECT nom_champ4 FROM nom_table2 WHERE condition);
B- Ordonner les données:

1- La clause ORDER BY permet de classer le résultat d'un select par ordre croissant (ASC) ou décroissant (DESC)

SELECT nom_champ1, nom_champ2, ...
FROM nom_table
ORDER BY nom_champ1 ; (ordre croissant par défaut)

SELECT nom_champ1, nom_champ2, nom_champ3, ...
FROM nom_table
ORDER BY nom_champ1, nom_champ2 DESC ;


2- AS  permet de préciser un nom de colonne.

SELECT nom_champ1, nom_champ2 AS nom_choisi
FROM nom_table;


Ce mot clef est très pratique pour donner un nom de colonne pour un résultat de calcul.

SELECT nom_champ1 + nom_champ2 AS total
FROM nom_table;
C- Les opérateurs ensemblistes

1- La clause GROUP BY  permet de subdiviser une table en plusieurs groupes. Chaque groupe regroupe l'ensemble des lignes ayant

SELECT nom_champ1, count(nom_champ2)
FROM nom_table
GROUP BY nom_champ1;


Cette requête donnera sur la colonne de gauche les différents nom_champ1 existants et dans la colonne de droite le total de nom_champ2 correspondant pour chaque nom_champ1.

2- la clause HAVING  permet de préciser la condition à laquelle le GROUP BY doit s'exécuter. Elle fonctionne de la même façon que la clause WHERE.

SELECT nom_champ1, count(nom_champ2)
FROM nom_table
GROUP BY nom_champ1
HAVING condition;
D- Les fonctions:

1- Fonctions et opérateurs arithmétiquespeuvent etre employés sur des noms de champ,

                    + addition
                    - soustraction
                    * multiplication
                              /  division

                             ABS (champ_numérique) valeur absolue
                             ROUND ( champ_numérique, nb) arrondi de champ_numérique à nb chiffres après la virgule
                             TRUNC ( champ_numérique, nb)  renvoit champ_numérique tronqué à nb chiffres après la virgule

2- Fonctions de chaines de caractères

    CONCAT( chaine1, chaine2 )

    INITCAP ( chaine) met le premier caractère de chaque mot de chaîne en majuscule et le reste du mot en miniscules

SELECT INITCAP (nom_champ)
FROM nom_table;


    LOWER  (chaine) met tous les caractères de la chaine en lettres minuscules.

    REPLACE (nom_champ,  ancienne_chaine, nouvelle_chaine) remplace dans le champ nom_champ, les  occurences d'ancienne_chaine par nouvelle_chaine).

    UPPER (chaine) met tous les caractères de la chaine en lettres majuscules.

3- Fonctions et opérateurs sur les dates
 

E- Les fonctions de groupe:

1- AVG: calcule la moyenne des valeurs contenues dans champ_numérique

SELECT AVG ( champ_numérique)
FROM nom_table;


2- COUNT: compte le nombre de lignes du résultat de la requête

SELECT COUNT(*)
FROM nom_table;


3- MIN : sélectionne la plus petite valeur contenu dans le champ nom_champ

SELECT MIN( nom_champ)
FROM nom_table;
4- MAX: sélectionne la plus grande valeur contenu dans le champ nom_champ
SELECT MAX( nom_champ)
FROM nom_table;
5- SUM: calcule la somme des valeurs de nom_champ
SELECT SUM( nom_champ)
FROM nom_table;


6- STDDEV: calcule l'écart-type des valeurs de nom_champ

SELECT STDDEV( nom_champ)
FROM nom_table;
7- VARIANCE: calcule la variance des valeurs de nom_champ
SELECT VARIANCE( nom_champ)
FROM nom_table;


Remarque: on peut combiner toutes ces fonctions avec l'attribut DISTINCT.
