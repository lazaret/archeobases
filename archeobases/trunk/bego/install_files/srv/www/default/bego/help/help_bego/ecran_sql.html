VI- LES ECRANS

8- L'écran SQL



L'écran SQL comme son nom l'indique sert d'interface entre la base de données et le langage de requêtes SQL. Il permet entre autres d'afficher les résultats sous forme de tableau à l'écran, de rediriger les résultats d'une requête vers un fichier texte et de stocker les requêtes elle-mêmes.


 


ECRIRE ET LANCER UNE REQUETE:

Taper la requête dans l'espace de saisie réservé à cet effet.

Saisir un nom de requête le plus explicite possible afin que d'autres utilisateurs puissent sans mal comprendre à quoi sert cette requête.

Choisir un type d'affichage: le type par défaut est Elaboré.

Simplifié: affichage à l'écran sous forme de colonnes (il est plus rapide que l'affichage élaboré)
2 enregistrements trouvés
zone   groupe   roche
-------------
  0              0             1
 
Elaboré: affichage à l'écran sous forme de tableau

 
Texte + Tabs: envoi du résulat de la requête dans un fichier, les colonnes de résultats seront separées par une tabulation. Le fichier aura cette forme:
zone    groupe  roche
0       0       '0'
0       0       '1'
Texte + Virgules: envoi du réultat de la requête dans un fichier, les champs résultats seront séparés par une virgule. Le fichier ressemblera à ceci:
zone,groupe,roche
0,0,'0'
0,0,'1'
Annuler une modification de la requête est encore possible en cliquant sur le bouton ANNULER.
Valider la requête en cliquant sur le bouton ENVOYER.


EDITER ET RELANCER UNE REQUETE DEJA SAUVEGARDEE:

 
Cliquer sur le lien Liste des requêtes


Sélectionner une requête en cliquant directement dans la colonne coderequête. La requête se lancera alors immédiatement.

On peut aussi lancer une requête en tapant directement son nom puis en cliquant sur ENVOYER.


APPELER L'AIDE SQL: cette aide est une présentation des différents mots clefs du langage de requête SQL et s'appuie sur des exercices corrigés.
